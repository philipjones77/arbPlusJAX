# Security Policy

arbPlusJAX is still establishing its full production security and supply-chain
process.

Until the full workflow lands:

- use repository issues only for non-sensitive security-hardening discussion
- do not post sensitive exploit details publicly before maintainers have had a
  chance to assess impact
- treat release provenance, dependency auditing, and artifact verification as
  active hardening work rather than already-complete guarantees

- [security_supply_chain_standard.md](/docs/standards/security_supply_chain_standard.md)
- [security_supply_chain_todo.md](/docs/status/security_supply_chain_todo.md)
