# Operational Support Standard

Status: active

## Purpose

This document defines the support and contributor-facing operational surfaces
required for a production library.

## Required Surfaces

The repository should maintain:

- contributor guide
- support/contact guide
- issue and bug triage guidance
- performance-regression triage playbook
- engineering checklist for adding a new family correctly

## Support Rule

Support, triage, and contributor instructions should be explicit repo surfaces,
not only maintainer folklore or issue-template assumptions.

## Required Evidence

The repo is compliant only if:

- the major support/governance entry files exist
- their relationship to standards, tests, and release process is documented
